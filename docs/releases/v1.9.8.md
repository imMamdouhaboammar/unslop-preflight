# Unslop Preflight v1.9.8

Release date: 2026-06-14

## Summary

v1.9.8 completes the rename to `unslop-preflight`, hardens strict gate behavior, and aligns the torture-bench feedback loop with the current rule IDs and scanner behavior.

This release keeps the product focused on one core promise: catch weak AI-generated frontend plans before a coding agent turns them into code.

## Highlights

### Package rename

The npm package is now `unslop-preflight` with two CLI aliases:

```bash
npx unslop-preflight autopilot
npx unslop autopilot
```

The repository, package metadata, homepage, bugs URL, and Socket badge now point to `imMamdouhaboammar/unslop-preflight`.

### Strict Gates Enforcement

All gates now run in strict mode. Warning and info guidance are treated as implementation blockers instead of soft suggestions.

This matters because AI coding agents often treat weak guidance as permission to proceed. Strict mode forces unresolved assumptions, weak specs, visual slop, and missing verification to be fixed before implementation.

### Torture Bench Alignment

The torture-bench work aligned benchmark expectations with canonical issue IDs and exposed remaining gaps in root-cause, modal scrollbar, z-index panic, and random typography detection.

The benchmark direction is now clear:

- use `issue.id` as the canonical scoring key
- avoid invented duplicate IDs
- preserve aliases only when needed for legacy expected files
- require critical misses to reach zero, not just a higher average score

### Validation hardening

The validation scripts now fail faster when weak guidance remains in the design handoff. This includes stricter handling for soft warnings, AI-slop words, and incomplete generated blocks.

### Heuristic fixes

This release includes focused fixes around:

- modal viewport guard matching
- overlay scanner matching for `overflow-y-scroll`
- auto-injected block stripping before rule checks
- idempotency across repeated audit/autopilot runs

## Why this release matters

Unslop is no longer only a markdown design checker. It is becoming a preflight layer for AI-built frontends.

The direction is now stricter and more testable: if the handoff is weak, the agent should not proceed. If the design hides root-cause gaps, visual slop, generic typography, unsafe overlays, or missing harness decisions, Unslop should block it before code is written.
