# Release Process

This repository supports manual GitHub releases with autogenerated release notes, npm publishing, and package-state checks.

## Files involved

| File | Purpose |
|------|---------|
| `.github/workflows/release.yml` | Manual workflow that creates the tag, GitHub release, generated notes, and optional npm publish |
| `.github/workflows/npm-publish.yml` | Package publish workflow for direct package-relevant pushes and tags |
| `.github/release.yml` | GitHub autogenerated release notes category configuration |
| `.github/RELEASE_TEMPLATE.md` | Manual release checklist and copy template |
| `CHANGELOG.md` | Human-maintained release history |
| `docs/NPM_PUBLISHING.md` | npm Trusted Publishing and token fallback setup |

## Create a release now

1. Open GitHub Actions.
2. Select `Release`.
3. Click `Run workflow`.
4. Leave `tag` empty to use `v<package.json version>`, or type a tag such as:

```text
v1.11.1
```

5. Keep `publish_npm` enabled if npm should receive the package.
6. Leave `prerelease` disabled for stable releases.
7. Run the workflow.

The workflow will:

1. read `package.json`
2. verify that the tag matches the package version
3. run `npm ci`
4. run `npm test`
5. run `npm pack --dry-run`
6. create the tag when missing
7. check whether npm already has the same version
8. publish to npm only if the version is new
9. create or update the GitHub Release with generated notes

## Autogenerated release notes

GitHub reads `.github/release.yml` to categorize pull requests in release notes.

Current categories:

- Breaking Changes
- Features
- Fixes
- Documentation
- CI and Release
- Maintenance
- Other Changes

Pull requests with `ignore-for-release` or `skip-changelog` are excluded.

## Release template

Use `.github/RELEASE_TEMPLATE.md` when writing manual notes or reviewing generated notes. It includes:

- release summary
- package links
- highlights
- verification checklist
- publishing checklist
- user install commands

## Version alignment checklist

Before release, confirm these match:

| Item | Expected |
|------|----------|
| `package.json` | current package version |
| `package-lock.json` | same current version |
| `README.md` badges | same current version |
| `CHANGELOG.md` | entry for current version |
| GitHub release tag | `v<package.json version>` |
| npm package | same version after publish |

## Notes

The release workflow publishes npm directly. This avoids relying on a tag created by `GITHUB_TOKEN` to trigger a separate workflow.

If npm Trusted Publishing is configured, the workflow can publish through OIDC. If it is not configured yet, add `NPM_TOKEN` as a repository secret.
